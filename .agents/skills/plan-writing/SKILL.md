# Structured Planning - Skill File

Derived from `antigravity-kit`. Focuses on breaking down work into small, verifiable chunks.

## 🎯 Task Breakdown Principles
- **2-5 Minute Tasks**: Each task should be atomic.
- **Max 10 Tasks**: If more, break into multiple plans.
- **Verification Every Step**: Every task MUST have a "Verify" criteria.
- **File Location**: Save plan as `{task-slug}.md` in the **PROJECT ROOT**.

## 📝 Plan Structure
```markdown
# [Task Name]
## Goal: [One sentence goal]
## Tasks:
- [ ] Task 1: Action [File/Line] -> Verify: [Check command/UI]
- [ ] Task 2: Action [File/Line] -> Verify: [Check command/UI]
## Done When:
- [ ] [Success criteria 1]
```

## ⚠️ Core Rule
NEVER assume a task is done without verifying it. If a plan is longer than 1 page, simplify it.
